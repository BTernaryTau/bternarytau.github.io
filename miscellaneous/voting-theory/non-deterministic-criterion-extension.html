Non-deterministic criterion extension

Non-deterministic criterion extension is a means of taking a criterion meant only for deterministic voting methods and creating a new version of it that can also apply to non-deterministic voting methods. This may be useful for making criteria robust to otherwise-deterministic voting methods that must occasionally resort to breaking ties randomly, as is standard behavior for many real-world elections. It is unclear to what extent this technique is useful for extending criteria that assume determinism to voting methods designed around randomness, such as random ballot.

Before this extension can be defined, some notation needs to be specified. In the following definition, {0,1}∞\{0, 1\}^\infty{0,1}∞ refers to the set of all possible infinite bitstrings, EEE refers to the set of all elections (where an election is just a list of ballots), and CCC refers to the set of all candidates.

For a given criterion, a voting method mmm passes the extended version of that criterion if there exists a function f:{0,1}∞×E→Cf : \{0, 1\}^\infty \times E \rightarrow Cf:{0,1}∞×E→C such that both of the following hold:

  1. For r∈{0,1}∞r \in \{0, 1\}^\inftyr∈{0,1}∞ chosen uniformly at random, Pr⁡[f(r,e)=c]=Pr⁡[m(e)=c]\Pr[f(r, e) = c] = \Pr[m(e) = c]Pr[f(r,e)=c]=Pr[m(e)=c] for all e∈Ee \in Ee∈E and c∈Cc \in Cc∈C.
  2. For all r∈{0,1}∞r \in \{0, 1\}^\inftyr∈{0,1}∞, the deterministic voting method mr=f(r,⋅)m_r = f(r, \cdot)mr​=f(r,⋅) passes the original criterion.

The idea here is that fff is essentially the same as mmm except with the randomness isolated to a single variable rrr. This makes it possible to ignore that randomness and compare the original criterion to fff via mrm_rmr​. At the same time, the extension only requires some function fff with the right properties to exist in order to prevent the extended criterion from being dependent on the internal workings of the voting method, specifically how it uses its randomness.

The input rrr is chosen from the set of infinite bitstrings in order to ensure that fff has an arbitrarily large number of random bits at its disposal. If for some reason this is insufficient, rrr could instead be drawn from a set with a greater cardinality, such as the power set of {0,1}∞\{0, 1\}^\infty{0,1}∞.
